# 前端 UI/UX 设计规范

## 1. 设计原则
- **简洁现代**：移除多余的装饰，使用扁平化设计，强调内容本身。
- **用户友好**：提供清晰的反馈（加载中、成功、失败），避免空白状态（Empty State）。
- **一致性**：统一使用 Naive UI 组件库，保持间距、颜色和字体的一致性。

## 2. 布局 (Layout)
- **全屏应用**：移除不必要的顶部留白，充分利用屏幕空间。
- **侧边栏**：保持侧边栏导航清晰，支持折叠。
- **内容区域**：
  - 统一内边距：`padding: 24px`
  - 响应式设计：适配不同窗口大小

## 3. 颜色系统 (Color System)
遵循 Naive UI 默认主题，并定义语义化用途：
- **主色 (Primary)**：用于主要操作按钮（如“上传并处理”）。
- **背景色**：
  - 侧边栏/Header：白色 (`#ffffff`) 或深色模式对应色。
  - 内容区背景：浅灰 (`#f5f7f9`) 以区分层级。
- **文本色**：
  - 标题：`#1f2225`
  - 正文：`#333639`
  - 辅助文本：`#80868b`

## 4. 组件规范
- **卡片 (Card)**：使用 `n-card` 包裹主要内容块，增加圆角和轻微阴影。
- **按钮 (Button)**：
  - 主要操作：`type="primary"`
  - 次要操作：`default` 或 `tertiary`
  - 危险操作：`type="error"`
- **空状态 (Empty)**：当没有数据时，必须展示 `n-empty` 并提供操作引导。

## 5. 交互反馈
- **加载状态**：异步操作（如读取文件）时，按钮应显示 `loading` 状态或使用全局加载条。
- **消息提示**：使用 `useMessage` 提供操作结果反馈。

## 6. 目录结构建议
- `views/`：页面级组件
- `components/`：可复用 UI 组件
- `stores/`：Pinia 状态管理 (推荐使用 `.ts` 而非 `.vue`)
